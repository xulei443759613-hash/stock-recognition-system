# Requirements

## 当前目标

构建一个面向新手个人投资者的 A 股群消息风控工作流。系统把群内“金股”“观察池”等消息转成结构化记录，自动核验价格、证据、盈亏比、仓位、一手风险，并输出可复盘的风控报告。

## 用户场景

1. 收到群消息后粘贴到 CLI，系统还原消息发出时价格。
2. 系统判断是否追高、是否超过目标、是否涨停、是否缺止损或目标。
3. 对 34,000 元账户按训练仓和一手风险输出是否可执行。
4. 输出关键价位、机会评级、训练档位、系统建议止盈止损和复盘任务。
5. 对 C 档和暂不实盘样本创建模拟观察，跟踪等待入场、模拟持仓、止盈、止损和顺序待查。
6. 后续用 `outcome` 记录 1/3/5/10 日表现，累计群源样本。

## 输入

- 群消息原文。
- 推送日期和时间。
- 可选手动行情：当前价、涨跌幅、量比、近 N 日收盘价。
- 可选证据核验：某条推荐逻辑 true/false。
- 账户金额，默认 34,000 元。

## 输出

- Markdown 风控报告。
- 动作信号：放弃、观察、等待回踩、模拟盘、小仓位试错等。
- 关键价位：入场、目标止盈、硬止损、系统建议止盈止损、可执行价。
- 训练档位：A 可实盘 100 股、B 轻仓训练 100 股、C 模拟观察、D 放弃。
- 模拟观察：纸面交易 ID、状态、模拟入场价、止盈价、止损价、更新记录。
- 证据采集计划。
- 复盘任务和群源评分。

## 硬边界

- 不自动下单。
- 不接真实券商交易。
- 不承诺收益率。
- 不把群消息、截图或老师话术作为买入证据。
- 不在缺少价格、目标、止损或出现硬性否决时输出真实买入动作。

## 验收标准

- 每条消息都能解析出股票代码或明确失败原因。
- 缺价格、超过目标、涨停、止损无效、盈亏比不足时必须拦截。
- 报告必须包含关键价位、核心原因、证据核验、机会评级、训练档位、系统建议止盈止损和复盘任务。
- 单元测试必须覆盖真实样本和硬性否决规则。
