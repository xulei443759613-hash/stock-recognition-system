# GitHub Skills 使用规范

GitHub 上的 skill、repo、notebook 可以用于：

- 群消息解析。
- 数据连接。
- 图表和报告。
- 记录和回测。
- 工作流自动化。

不能用于：

- 直接采纳买卖信号。
- 接入真实券商自动交易。
- 输入未审查代码的 API key。
- 用 star 数证明策略有效。

接入前检查：

1. README 和许可证。
2. 数据来源。
3. 是否包含自动交易代码。
4. 是否存储凭据。
5. 是否有隐藏网络请求。
6. 先用样例消息测试。

## 当前适合接入的方向

- GitHub Actions：已接入，用来保证规则和解析不被改坏。
- Tushare data skill：适合后续补财务、公告、行情结构化数据，但需要 token，不能硬编码凭据。
- AkShare：适合做 A 股公开数据补充，但必须记录数据来源和更新时间。
- pandas/NumPy：适合先做复盘样本统计，是当前最轻量、最有用的分析能力。

## 当前暂缓

- 自动交易和券商下单库。
- 只提供荐股结果、不提供数据来源的仓库。
- 大型量化框架直接接入生产流程；等复盘样本足够后再做研究分支。
