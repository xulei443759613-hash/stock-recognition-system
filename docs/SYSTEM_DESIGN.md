# 股票识别系统设计

## 目标

把群内股票消息变成结构化风控分析：

1. 解析股票代码、入场价、目标价、止损价和推荐逻辑。
2. 检测红线话术和营销诱导。
3. 补全行情、公告、财务、资金、调研和新闻证据。
4. 计算盈亏比和新手仓位上限。
5. 输出非投资建议的动作信号。
6. 每日记录并复盘群源质量。
7. 生成条件式入场计划、止损止盈计划和新手仓位上限。

## 模块

- `parser.py`: 群消息解析。
- `rules.py`: 红线、硬否决、评分。
- `risk.py`: 盈亏比和最大亏损股数。
- `engine.py`: 决策编排。
- `data_sources.py`: 数据源接口，后续接 AkShare/Tushare/CNINFO。
- `records.py`: 每日记录。
- `reporting.py`: Markdown 风控分析报告。
- `technical.py`: 技术面体检，识别过热、走弱、波动过大。
- `followup.py`: 复盘任务生成、保存和读取。
- `cli.py`: 命令行入口，适合新手粘贴群消息后生成报告。

## 硬性规则

- 当前价超过目标：不追。
- 当前价未知：只做条件分析。
- 止损缺失或无效：不做真实仓位。
- 盈亏比低于 1.5：新手放弃。
- 涨停后才看到：不追。
- 当前价已到或跌破止损：不做反弹幻想。
- 目标价不高于止损价：价格结构无效。

## 决策流程

1. 解析群消息。
2. 检测红线和硬性否决。
3. 核验推荐逻辑证据。
4. 结合消息发出时价格、当前价、入场区间、短期涨幅、大盘和板块判断时机。
5. 检查 5 日/20 日走势、均价结构、振幅和量比。
6. 计算当前价、消息时价格和入场区间的盈亏比。
7. 输出动作信号、入场条件、止损止盈、仓位上限。
8. 写入每日记录，自动生成 1 日、3 日、5 日、10 日复盘任务。

## 新手保护

- 默认不输出无条件买入。
- 缺当前价时只观察。
- 数据源失败时降级，不用猜测补数据。
- 单笔最大亏损默认控制在账户 1%。
- 真实仓位必须同时满足价格、证据、盈亏比和止损条件。
- 系统反推满足最低盈亏比的最高买入价，超过该价格不做。
- 技术面过热或走弱时，至少降级为模拟盘或观察。
- 复盘任务未完成时，不评价群源优劣。

## 工程稳定性

- `tests/`: 固定关键样本和硬性否决规则。
- `.github/workflows/ci.yml`: 推送和拉取请求时运行语法检查和单元测试。

## 群源复盘

至少累计 20 条样本后再评价群源：

- 达到目标比例。
- 触发止损比例。
- 尾盘推送比例。
- 超过目标或涨停后推送比例。
- 系统放弃比例。
