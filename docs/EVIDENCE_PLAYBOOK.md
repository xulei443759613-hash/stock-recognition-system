# Evidence Playbook

这个文档定义收到群消息后要自动采集和人工补充的证据。系统原则是先验证事实，再看价格和风险；没有证据不升级仓位。

## 基础 P0 证据

每条消息都必须先过三类基础证据：

1. 消息时点价格：推送那一分钟或此前最近一分钟的价格、涨跌幅、是否涨停、是否已经超过目标价或入场上沿。
2. 20日价格结构：最近20个收盘价、5日/20日涨跌幅、量比、换手率，主要用于过滤追高和破位。
3. 账户承受力：100股成本、100股到止损的亏损额、是否超过训练仓和单笔亏损上限。

任何 P0 证据缺失或触发否决，都只能输出观察、等待或放弃。

## 群消息逻辑到证据类型

| 群消息说法 | 证据类型 | 优先级 | 首选来源 | 通过标准 | 否决标准 |
| --- | --- | --- | --- | --- | --- |
| 基金参与、社保加仓、机构参与 | 机构持仓 | P1 | 巨潮资讯定期报告、交易所公告 | 最近一期正式披露显示持有或增持 | 无正式披露、数据过旧、只有截图 |
| 游资、高控盘、控盘、资金热度、主力 | 交易行为 | P0 | 交易所龙虎榜、成交额、换手率、量比 | 最多证明短线活跃，且价格仍合规 | 没有龙虎榜证据、已追高、用控盘包装不可验证结论 |
| 盈利拐点、毛利率上升、现金流提升、收入稳健 | 财务质量 | P1 | 巨潮资讯财报、交易所公告、公司财报原文 | 最近两期指标改善，现金流不背离利润 | 一次性收益、现金流恶化、财报不支持 |
| 行业内高评级、研报、目标价 | 外部评级 | P2 | 券商研报原文、合规数据商 | 机构、日期、评级、假设可核实 | 二手摘录、研报过旧、无法和公告财报交叉验证 |
| 材料、先进封装、涨价、题材 | 行业题材 | P2 | 公司公告、行业协会、权威价格指数、可信财经媒体 | 公司确有业务暴露，事件可确认 | 只有概念标签、收入占比低、价格已充分反映 |
| 机构调研 | 调研记录 | P1 | 巨潮资讯投资者关系记录、互动易、上证e互动 | 公开记录可查，问答要点匹配 | 无公开记录，包装内幕或确定性收益 |

## 当前阶段不采信的内容

- 群内“老师”口头解释。
- 模糊的“控盘程度极高”。
- 只给截图、不提供来源链接的数据。
- “不构成投资建议”后又给入场、目标、止损的营销闭环。
- 用收益目标倒推出买入结论。

## CLI

收到消息后可先跑：

```powershell
python -m stock_recognition_system.cli evidence-plan --message-file examples/group_message.txt
```

完整分析仍使用：

```powershell
python -m stock_recognition_system.cli review `
  --message-file examples/group_message.txt `
  --push-date 2026-07-01 `
  --push-time 10:57 `
  --auto-market-data `
  --history-days 20 `
  --account-value 34000
```
