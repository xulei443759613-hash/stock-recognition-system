# Product Roadmap

## 产品定位

本项目是新手个人投资者的股票群消息风控工作台，不是荐股系统、预测系统或自动交易系统。

核心目标：

1. 把群消息变成结构化记录。
2. 防止追高、涨停追入、尾盘被带节奏。
3. 用当前价、证据、盈亏比、止损和仓位共同约束动作。
4. 连续复盘群源质量，而不是凭一两次涨跌判断。

## 产品北极星指标

- 冲动交易拦截率：系统判定放弃/观察后，用户没有追高的比例。
- 规则覆盖率：每条消息是否都有价格、证据、盈亏比、仓位和复盘任务。
- 复盘完成率：1 日、3 日、5 日、10 日复盘是否按时补齐。
- 群源样本量：单个群源至少 20 条后才评价质量。
- 重大风险漏判数：超过目标、涨停、止损无效、缺当前价是否被正确拦截。
- 可执行错失率：系统未给真实仓位后，价格曾回到可执行价再触达目标的比例。
- 非可执行上涨率：系统放弃后触达目标，但中途没有给出合理买点的比例。

## 优先级

### P0 稳定性和风控底座

- 自动测试覆盖硬性否决规则。
- GitHub Actions 自动跑语法检查和单元测试。
- 每条样本必须可复盘。
- 所有数据缺失都降级，不用猜测补事实。

### P1 数据可信度

- 接 AkShare/Tushare 获取行情、历史 K 线、换手率、量比。
- 接巨潮资讯和交易所公告核验公告、财报、调研。
- 所有外部数据必须写入来源和时间。
- 支持“消息发出时价格”与“看到消息时价格”两个口径。

已落地：

- 腾讯分时分钟线可按 `push_date/push_time` 自动还原消息发出时价格。
- 消息时价格会同步给出当时涨跌幅和涨停状态，避免被次日行情污染。

### P2 决策能力

- 自动反推满足最低盈亏比的最高买入价。
- 输出机会评级，区分真实仓位、模拟跟踪、等待更优价格、补证据观察和剔除机会。
- 识别连续涨停、炸板、高位放量、尾盘异动。
- 加入大盘、行业、题材热度和个股流动性过滤。
- 输出“放弃原因排名”，帮助新手学习。

已落地：

- 风控报告新增“机会评级”，在不放松真实仓位约束的前提下保留观察机会。
- 自动计算普通风控最高买入价、短线最高买入价、100 股止损风险最高买入价和训练模式综合可执行价。
- 对盈亏比不足但仍有观察价值的样本，输出需要回撤多少才重新评估。

### P3 复盘和群源评分

- 记录 1/3/5/10 日表现。
- 自动判断是否达到目标、触发止损、次日高开低走。
- 对群源输出样本数、追高率、尾盘率、止损率、有效率。
- 区分“可执行错失机会”和“非可执行上涨”，避免放弃后上涨就误判系统过保守。
- 样本少于 20 条只观察，不评价优劣。

已落地：

- `outcome` 命令：把复盘结果追加到 `records/outcomes.jsonl`。
- `source-score` 命令：按来源统计样本数、触达目标率、止损率、尾盘率和追高率。
- 价格字段可自动推断触达目标、触发止损和超过目标后追高。
- `outcome` 命令会输出机会复盘分类：可执行错失、非可执行上涨、顺序待查或未触达目标。

### P4 用户体验

- 提供本地 Web 页面录入群消息。
- 支持粘贴截图后人工确认识别结果。
- 报告可导出 Markdown/HTML。
- 新手模式只显示结论、原因和下一步。

## 不做的事

- 不自动下单。
- 不接真实券商交易。
- 不把群消息或新闻当作 A 级证据。
- 不用单次成功案例证明策略有效。
- 不承诺收益率。

## 下一步执行

1. 继续补测试，固定每条已知样本的输出。
2. 增加“消息发出时价格”字段和数据来源说明。
3. 建立真实样本库，先积累 20 条。
4. 用可执行错失率决定是否放宽机会评级阈值。
5. 再接更多公告和财务数据源。
